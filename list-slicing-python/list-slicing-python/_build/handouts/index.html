Intro

This exercise will have you implement various utility functions to use with lists. There’s one catch though — you can only use fundamental list operations to complete each function!

We’ve included a file — list_operations.py — that contains function stubs (function stubs are functions that are partially complete). Each function has a docstring that describes how it should work and gives an example of how the function ought to work:

def head(input_list):
    """Return the first item of the input list.

    For example:

      >>> head(['Jan', 'Feb', 'Mar'])
      'Jan'
    """

    return None

Note that the code examples show how each function should work if you were running them in the Python interpreter. That’s why they begin with >>>. These examples aren’t purely illustrative though; Python has the ability to do something pretty rad with them. We’ll show you what we mean in the next section.

Runnning list_operations.py also runs 14 tests (and they all failed but that’s understandable — the functions are incomplete after all). These tests actually come from the examples in those docstrings! As such, they’re aptly known as doctests.

When you run this file, the doctest code examples are treated like real code and run inside the Python interpreter. Depending on what happens when the code samples get run, Python’s doctest utility formats some helpful output for you, which is what you’re seeing when you run the file.

All of your tests failed because there’s nothing in any of your list_operations.py functions. So whatever your functions return (they return nothing, or the wrong result) doesn’t match what the tests expect (very specific results, including not nothing). When a test fails, it will print the characters that don’t match the expected printout in the terminal. This will be helpful for troubleshooting your functions.

Fundamental Operations On Lists

Anything that you can’t re-implement in Python is fundamental to the language. Fundamental list operations include:

  • List indexing (some_list[index])

  • List indexing assignment (some_list[index] = value)

  • List slicing (some_list[start:end])

  • List slicing assignment (some_list[start:end] = another_list)

  • List index deletion (del some_list[index])

  • List slicing deletion (del some_list[start:end])

Not Allowed!

You may not:

  • Use any of the Python built-in methods for lists.

  • Use the len() function Python provides to get the list length.

Go!

Your mission: Make all of the tests pass.

You’ll do this by actually writing the functions in list_operations.py.

IMPORTANT: Each time you think you have a function working, re-run your program and check whether the number of failures went down. Because of how we set up the files, all you have to do is:

$ python3 list_operations.py

You can also read the output and double check that there are no failures showing for the function that you were writing.

When you’ve succeeded, you should see only this output when you run your tests:

$ python3 list_operations.py
ALL TESTS PASSED

Run this command and check the output after every single function that you write. The tests are meant to help you.
