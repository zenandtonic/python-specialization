Roll Your Own List Methods

Your task is to implement functional versions of the standard list methods. Here, functional doesn’t describe something that works. Functional describes code that is written in a functional programming style. Functional programming is a method for writing code where (amongst other things) there are only functions and no methods.

Again, you can only use the fundamental list operations that we mentioned in the main part of the lab. This time, we’ll also allow you to use for-loops like the following:

for item in some_list:
    # ... etc.

Each function imitates a built-in list method, as described by the docstring for each function. Play with the built-in methods in the Python console to get a feel for how they work before trying to write your custom version. Use the doctests in each function for concrete examples of expected behavior.

As with the main exercise, type the following to run the tests:

$ python3 further_study.py

Before starting, you should see that there are 12 failing doctests. When you’ve successfully completed the further study, you should see:

$ python3 further_study.py
ALL TESTS PASSED
