Rubric

topics

Lists, slicing, testing, in place vs copy in memory

further study concepts

N/A

requires

Basic Python and Command Line, Lists lecture

time

2-4 hours

authors

Christian Fernandez, Joel Burton, Meggie Mahnken

Goals

Students will complete functions in order to pass tests working with lists and testing. Students will gain familiarity with Python functions, list slicing and testing.

Students will be exposed to testing practices.

Gotchas

There is confusion on inner_four_end function, the code is correct, it is asking for the list in the correct order, but the docstring could be interpreted as reversed order

When doing code reviews, watch for:

  • make sure students are actually USING the test output to check whether their functions succeed.

    • If they haven’t yet run the file and examined the test output, ask if they have read all of the directions.

  • use of built-in functions that are not allowed (append, reverse, etc.)

  • editing the list in-place vs. making a copy

  • most function in part 1 can be done in one line

  • handles more than one space between operators or numbers

  • for the function that deletes two items, point out that it’s better to delete the latter item first due to index change

  • replace_third_and_last: can be done on one line using unpacking

    • input_list[2], input_list[-1] = 37, 37

    • Have also seen: input_list[2] = input_list[-1] = 37

Notes

Important note: in the main part of the exercise, students CAN use

del input_list[3]

However, you could easily accomplish this without using del with

input_list[3:4] = []

This exercise can take a while. Be prepared to reassure students who don’t make it through the extra credit.

Code Reviews

Code Review 1

  • list_operations.py file suggests students get a code review in the middle of the file

  • Will have completed the following functions:

    • head (index to get first item)

    • tail (return all elements except the first item of a list)

    • last (return the last item of a list)

    • init (return all elements except last item from a list)

  • Good time to discuss pairing – who’s driving, who’s navigating?

Code Review 2

  • Students should have all tests passing in the list_operations.py file.

  • Discuss what doctests are and how they’re implemented.

  • Discuss different ways of deleting (del vs. list slicing)

    • del input_list[index]

    • input_list[start:end] = []

  • Discussion: are students editing list in place or making copies? What does it mean to do both?

  • Shouldn’t have used any built-in list methods

Further Study

  • Not many students got to this piece in Grace Summer 2016

  • Almost all pairs got to this in Grace Fall 2016 (every pair except one)

  • Students build their own list methods using slicing and indexing

  • Functions should build off of one another (ex: students can use custom_len in their custom_append function…)
